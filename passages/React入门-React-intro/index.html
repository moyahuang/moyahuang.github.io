一、 React是什么

React是一个声明式的组件化构建用户界面的JavaScript库。

React一般用于内容随着时间变化的大型Web应用以及单页应用程序（例如Instagram），对其进行视图渲染。

（React诞生的背景原因：Facebook认为MVC模式他们巨量的代码库和庞大的组织使得每需要添加一项新功能或者特性时，系统的复杂度成级数增长，致使代码变得脆弱而不可预测。为了以“某种方式组织代码，使其更加可预测”）

国内大型互联网企业几乎都在使用React，诸如阿里、腾讯、滴滴、华为等等。

四、React的优点

  1. 组件化

    组件化设计思想的原则一是在于单一职责，即一个组件只做一件事情。其好处在于修改组件时，不用担心会对其他组件造成影响。但是过度拆分组件，没有必要，反而会增加编写繁琐程度。因此如果某个功能集合会发生潜在的改变，就应该最大程度地保证该功能集合的单一职责性。

    举个例子：某个组件需要通过网络请求获取数据并展示数据内容，其潜在的功能集合改变有：

    • 请求API地址发生变化
    • 请求返回数据格式变化
    • 开发者想更换网络请求第三方库，比如jQuery.ajax改成axios
    • 更改请求数据逻辑
  1. 数据状态管理使用单项数据流，查找问题快捷、方便

单向数据流是指在React中数据只能由父组件通过props传递给子组件，不能由子组件向父组件传递数据。
