what is THIS

What THIS is depends on how a function is called in most cases, though in ES5, bind() can be used to set the value of THIS.

In non-strict mode, THIS is always a reference to an object which is the current execution context. In strict mode, THIS can be any value(:question: which I dont really understand)

Basically the execution context can be divided into global and function context(whether it be an object method or not)

Global context

In the global execution context (outside of any function), THIS refers to the global object. In a browser, THIS is window; however, in Node, THIS is module.exports(:question: I am not sure if it is the global object).

In web browser, the global object is window

In Node, global is the global object but not THIS. THIS refers to module.exports.

:question:If I define the num as a property of num, we can see from the above and the following example that THIS is not global.

Function context

Inside a function, THIS is set by how the function is called.

In non-strict mode, THIS defaults to the global object.

:question: This is SOOO weird! In terms of the Node case: when in a function, THIS refers to global; when outside a function, THIS refers to module.exports.

In strict mode, THIS remains as undefined unless it is set by using call()or apply()
